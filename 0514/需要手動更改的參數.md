你最常需要改的是這四個地方：

## 3. probe-to-gene 流程：改這裡

這個決定你是「先合併 probe 再跑 limma」，還是「先跑 limma 再合併 probe」。

```r
COLLAPSE_STAGE <- "post_limma"
```

可選：

```r
COLLAPSE_STAGE <- "post_limma"
```

或：

```r
COLLAPSE_STAGE <- "pre_limma"
```

我建議你目前先用：

```r
COLLAPSE_STAGE <- "post_limma"
```

因為比較接近 GEO2R 的邏輯：先在 probe-level 做分析，再整理成 gene-level。

---

## 4. probe-to-gene 方法：改這裡

這就是你問的 method：

```r
PROBE_TO_GENE_METHOD <- "min_pvalue"
```

可以改成這幾種：

```r
PROBE_TO_GENE_METHOD <- "min_pvalue"
```

意思是：同一個基因有多個 probe 時，保留 **P.Value 最小** 的 probe。

```r
PROBE_TO_GENE_METHOD <- "min_adj_pvalue"
```

意思是：保留 **adj.P.Val 最小** 的 probe。

```r
PROBE_TO_GENE_METHOD <- "max_abs_logfc"
```

意思是：保留 **|logFC| 最大** 的 probe。

```r
PROBE_TO_GENE_METHOD <- "max_mean"
```

意思是：保留 **平均表現量最高** 的 probe。

---

## 建議你怎麼跑

你可以先跑這組：

```r
COLLAPSE_STAGE <- "post_limma"
PROBE_TO_GENE_METHOD <- "min_pvalue"
CUTOFF_LOGFC <- 2.0
```

然後第二次只改 method：

```r
PROBE_TO_GENE_METHOD <- "max_abs_logfc"
```

再跑一次。

如果要回到你原本 Gemini 那版 Max Mean，就改成：

```r
COLLAPSE_STAGE <- "pre_limma"
PROBE_TO_GENE_METHOD <- "max_mean"
```

注意：
如果你用：

```r
COLLAPSE_STAGE <- "pre_limma"
```

那 method 只能是：

```r
PROBE_TO_GENE_METHOD <- "max_mean"
```

因為 `P.Value`、`adj.P.Val`、`logFC` 都是 limma 跑完後才會產生，所以 pre-limma 階段不能用 `min_pvalue` 或 `max_abs_logfc`。
